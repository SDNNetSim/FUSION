Common Next Steps by Role
==========================

For Students
------------

**If you're taking a course on optical networks:**

1. Complete :doc:`../concepts/optical_networking_basics` through :doc:`../concepts/resource_allocation`
2. Work through :doc:`../examples/basic_simulation`
3. Run experiments comparing different algorithms
4. Analyze and present results

**Project Ideas:**

* Compare routing algorithms on different topologies
* Evaluate spectrum assignment strategies
* Analyze impact of traffic patterns
* Study network performance under failures

For Researchers
---------------

**If you're conducting research in optical networking:**

1. Review :doc:`../user_guide/running_simulations` and :doc:`../user_guide/configuration_reference`
2. Explore :doc:`../examples/batch_simulations` for experimental methodology
3. Study :doc:`../api/index` to understand extensibility
4. Read :doc:`../developer/extending` to add custom algorithms

**Typical Workflow:**

1. Design experiments (topologies, traffic, algorithms)
2. Create configuration files
3. Run batch simulations
4. Analyze results statistically
5. Create publication-quality plots
6. Cite FUSION in your papers

For Developers
--------------

**If you're developing new algorithms or features:**

1. Complete :doc:`../developer/development_setup`
2. Study :doc:`../developer/architecture`
3. Read :doc:`../developer/extending`
4. Follow :doc:`../developer/coding_standards`
5. Review :doc:`../developer/testing`
6. See :doc:`../developer/contributing` to share your work

**Development Workflow:**

1. Fork FUSION repository
2. Create feature branch
3. Implement and test your feature
4. Run pre-commit hooks
5. Submit pull request

For Industry Professionals
--------------------------

**If you're evaluating network designs or technologies:**

1. Review :doc:`../user_guide/running_simulations`
2. Study :doc:`../examples/custom_topology` to model your networks
3. Explore :doc:`../examples/performance_optimization` for large-scale studies
4. Use :doc:`../user_guide/visualization` for reporting

**Use Cases:**

* Capacity planning
* Technology evaluation (e.g., comparing vendors)
* Network upgrade analysis
* Performance benchmarking
