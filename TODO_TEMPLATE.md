## Template Notes (Remove in actual TODO)

### Priority Guidelines:
- **High Priority**: Blocking issues, critical bugs, security concerns
- **Medium Priority**: Usability improvements, feature enhancements
- **Low Priority**: Nice-to-have features, optimizations

### Item Structure:
1. **Issue**: What's the problem? (1-2 sentences)
2. **Files**: Specific file references when relevant
3. **Impact/Missing**: Why does this matter?
4. **Solution**: How to fix it (concise, no detailed steps)

### Writing Guidelines:
- **Concise**: Each item should be scannable in 30 seconds
- **Actionable**: Clear problem statement and solution direction
- **Specific**: Reference exact files and line numbers when possible
- **No Implementation Details**: Keep solutions high-level
- **Remove Completed Items**: Don't track completed work in TODO

### Optional Sections (add if relevant):
- **Research Items**: For unclear requirements needing investigation
- **Dependencies**: Items blocked by external factors
- **Breaking Changes**: Items that require API changes

### Maintenance:
- Remove completed items immediately
- Move items between priorities as context changes
- Reference other TODO files when items span modules
- Keep total items under 15 for readability
